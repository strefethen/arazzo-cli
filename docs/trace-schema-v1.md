# Trace Schema v1

`trace.v1` is the stable on-disk execution trace format generated by:

```bash
arazzo --json run <spec> <workflow-id> --trace <path>
```

## Top-level object

- `schemaVersion` (string): must be `"trace.v1"`
- `tool` (object): emitting tool metadata
- `run` (object): run-level metadata and status
- `inputs` (object): sanitized workflow inputs
- `steps` (array): ordered step attempt records

## `tool`

- `name` (string)
- `version` (string)

## `run`

- `specPath` (string)
- `workflowId` (string)
- `parallel` (boolean)
- `dryRun` (boolean)
- `timeoutMs` (number)
- `startedAt` (RFC3339 string)
- `finishedAt` (RFC3339 string)
- `durationMs` (number)
- `status` (string: `success` or `failure`)
- `error` (string, optional on failure)

## `steps[*]`

- `seq` (number): deterministic per-run sequence
- `workflowId` (string)
- `stepId` (string)
- `attempt` (number)
- `kind` (string: `http` or `workflow`)
- `operationPath` (string, optional)
- `workflowIdRef` (string, optional)
- `durationMs` (number)
- `request` (object, optional)
- `response` (object, optional)
- `criteria` (array, optional)
- `decision` (object)
- `outputs` (object, optional)
- `error` (string, optional)

## Compatibility guarantees

1. `schemaVersion` is the compatibility gate.
2. Existing fields keep meaning and type within `trace.v1`.
3. New fields may be added (forward-compatible).
4. Breaking changes require a new schema version.
